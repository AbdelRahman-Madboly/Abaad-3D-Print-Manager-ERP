`src/core/config.py`'s `DEFAULT_SETTINGS` dict is missing keys that
`setup_wizard.py` references at runtime. Add these or it crashes on first run:

```python

    # existing keys (keep as-is)
    "company_name":          COMPANY["name"],
    "company_phone":         COMPANY["phone"],
    "company_address":       COMPANY["address"],
    "default_rate_per_gram": str(DEFAULT_RATE_PER_GRAM),
    "next_order_number":     "1",
    "deposit_percent":       "50",
    "quote_validity_days":   "7",
    # ADD these:

    "currency_symbol":       "EGP",
    "company_logo_path":     "",
    "setup_complete":        "0",
    "company_tagline":       COMPANY["tagline"],
    "company_social":        COMPANY["social"],
}
```

Also remove `OLD_JSON_DB` and `OLD_USERS_JSON` — these are Phase 2 leftovers
that should have been deleted. Confirm nothing else imports them first
(`grep -r "OLD_JSON_DB\|OLD_USERS_JSON" src/ tests/`).

### Fix 2 — `main.py`: wire the setup wizard

`run_setup_wizard_if_needed` exists in `setup_wizard.py` but is never called
from `main.py`. Add it to the boot sequence between steps 3 (auth) and 4
(login loop):

```python
# 3b. First-run setup wizard (shows once if setup_complete != "1")
from src.ui.dialogs.setup_wizard import run_setup_wizard_if_needed
run_setup_wizard_if_needed(db, root)
```

Note: `root` is created inside the login loop currently. You may need to
create a temporary hidden root before the loop for the wizard, or restructure
slightly. Document your approach in the report.

### Fix 3 — `app.py`: replace hardcoded branding strings

Two places still hardcode `"Abaad ERP"` as a literal (not reading from
settings or config):

1. Header label (line ~100): `tk.Label(hdr, text="Abaad ERP", ...)`
   → Replace with: read `company_name` from `db.get_setting("company_name")`
   with fallback to `config.COMPANY["name"]`

2. Status bar version label: `text=f"Abaad ERP v{APP_VERSION}"`
   → Replace with: `text=f"{APP_TITLE}"` (APP_TITLE is already
   `f"{APP_NAME} v{APP_VERSION}"` from config — use that constant)

The product name (`APP_NAME = "Abaad ERP"` in config) stays — that's the
correct Abaad-as-product identity. Only the header company name (the *tenant*
identity) needs to read from settings.
