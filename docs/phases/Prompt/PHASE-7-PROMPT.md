Report exactly what you find before making any changes.

### Task A2 — Fix any PDF issues found in A1

Common issues to expect and fix:
- **Missing footer:** add `config.PDF_FOOTER_CREDIT` in small grey text
  at the bottom of every page
- **Logo not loading:** ensure the fallback to `config.LOGO_PATH` works
  when `company_logo_path` setting is empty or file doesn't exist
- **Layout overflow:** long text (customer names, addresses) must wrap, not
  overflow off the page
- **Missing page numbers:** add "Page N of M" to footer alongside credit
- **Hardcoded currency:** `generate_text_receipt()` hardcodes `"EGP"` twice.
  Replace with `db.get_setting("currency_symbol") or "EGP"`.
- **`default_cost_per_gram` not in DEFAULT_SETTINGS:** the setup wizard (Phase 2)
  saves this key to the `settings` table, but `DEFAULT_SETTINGS` in `config.py`
  has no entry for it. Any cost-calculation code that reads this key must use:
  ```python
  cost = db.get_setting("default_cost_per_gram")
  if cost is None:
      cost = str(config.DEFAULT_COST_PER_GRAM)
  ```
  Add `"default_cost_per_gram": str(DEFAULT_COST_PER_GRAM)` to `DEFAULT_SETTINGS`
  in `config.py` so all callers can rely on `get_setting()` returning a value.

Footer pattern for reportlab:
```python
def _draw_footer(self, canvas, doc):
    """Draw footer on every page — credit + page number."""
    canvas.saveState()
    canvas.setFont("Helvetica", 8)
    canvas.setFillColorRGB(0.5, 0.5, 0.5)  # grey
    canvas.drawString(
        doc.leftMargin,
        0.5 * 72,  # 0.5 inch from bottom
        config.PDF_FOOTER_CREDIT
    )
    canvas.drawRightString(
        doc.pagesize[0] - doc.rightMargin,
        0.5 * 72,
        f"Page {doc.page}"
    )
    canvas.restoreState()
```

## Part B — Code Documentation

### Task B1 — Module docstrings

Every `.py` file in `src/` must have a module-level docstring describing:
- What the module is
- Its place in the architecture
- Key classes/functions it exports

Example (already done in some files, verify all):
```python
"""
src/services/order_service.py
=============================
Business logic for order lifecycle management.

All SQL lives in DatabaseManager — this service converts raw dicts to
Order dataclasses, applies business rules, and orchestrates cross-service
operations (e.g. recording print jobs on status transitions).

Key class: OrderService
"""
```

### Task B2 — Class and method docstrings

Every public class and every public method must have a Google-style
docstring. Use `src/services/inventory_service.py` as the style reference
(it's already well-documented).

Format:
```python
def get_orders_needing_attention(self) -> dict:
    """Return orders that need admin attention.

    An order needs attention if it is:
    - In "Ready" status and hasn't been updated in more than
      config.ORDER_READY_ALERT_DAYS days, OR
    - In "Ready" or "Delivered" status with outstanding payment
      (amount_received < total_price)

    Returns:
        dict with keys:
            "ready_overdue": list of Order — idle Ready orders
            "payment_due":   list of Order — unpaid Ready/Delivered orders
    """
```

Files to prioritize (do all, but these are highest-value):
1. `src/core/database.py` — every method
2. `src/core/models.py` — every dataclass and its fields
3. `src/services/*.py` — every public method
4. `src/auth/auth_manager.py` — every public method
5. `src/ui/app.py` — class docstring + major methods
6. `src/ui/tabs/*.py` — class docstrings at minimum

### Task B3 — Inline comments for non-obvious logic

Add `# why` comments (not `# what`) for any logic that isn't obvious from
reading the code. Examples:

```python
# Phase 1: using shared-cache + keepalive for :memory: because each
# method opens/closes its own connection — a plain :memory: DB would
# be destroyed between calls
conn_str = "file::memory:?cache=shared&mode=memory" if self._memory else str(self._path)
```
