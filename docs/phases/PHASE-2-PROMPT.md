# Phase 2 — Tenant Brand System
> **Type:** feature + refactor. This is the core product promise.
> **Session start:** `cat docs/CLAUDE.md` then this file.
> **Prerequisite:** Phase 1 complete, all tests green.
> **Branch:** `feature/phase-2-tenant-brand` off `develop`

---

## Goal

Make "any 3D-printing business can deploy this fresh" actually true.
Every hardcoded piece of company identity moves to the `settings` SQLite table.
The first-run wizard is extended to capture a complete initial setup.
The app header, login screen, and all generated PDFs show the tenant's brand.

---

## The two-identity rule (read before touching anything)

```
PRODUCT  = Abaad (the app/maker)  → stays in config.py → NEVER change per customer
TENANT   = the business owner     → lives in settings table → configured at first run
```

Correct: window title says "Abaad ERP v5.0" (product)
Correct: header shows "Al-Nour 3D Printing" + their logo (tenant)
Correct: PDF footer says "Generated by Abaad ERP" (product credit)
Correct: PDF body shows tenant name, logo, phone, address

### Task 2 — Extend the setup wizard to 4 steps

Current wizard (Step 1): company name, address, phone, tagline, social,
app subtitle, currency, rate per gram, logo.

**Add Step 2 — Initial filament inventory:**

Show the default color list from `config.DEFAULT_COLORS`. For each color,
a number input (weight in grams, default 0). User enters how much of each
color they currently have in stock. On save, call
`InventoryService.add_spool()` for each non-zero entry, creating one spool
record per color with that weight.

"Skip" on this step leaves inventory empty (fine — they can add later).

**Add Step 3 — First printer:**

Fields with pre-filled defaults from `config.py`:
- Printer name (default: `"Printer 1"`)
- Purchase price (default: `config.DEFAULT_PRINTER_PRICE`)
- Expected lifetime kg (default: `config.DEFAULT_PRINTER_LIFETIME_KG`)
- Electricity rate EGP/hr (default: `config.ELECTRICITY_RATE`)
- Nozzle cost (default: `config.NOZZLE_COST`)
- Nozzle lifetime grams (default: `config.NOZZLE_LIFETIME_GRAMS`)

On save, call `PrinterService.add_printer()`.
"Skip" is fine — they can add printers in the Printers tab later.

**Add Step 4 — Cost defaults review:**

Show the current business defaults and let the user confirm or adjust:
- Rate per gram (selling price) — pre-filled from step 1 or config default
- Cost per gram (material cost) — pre-filled from `config.DEFAULT_COST_PER_GRAM`
- Deposit % — pre-filled from `DEFAULT_SETTINGS["deposit_percent"]`
- Quote validity days — pre-filled from `DEFAULT_SETTINGS["quote_validity_days"]`

Save all to settings table. Show a "Finish Setup" button.

**Multi-step navigation:** use a simple step indicator (e.g. `"Step 2 of 4"`)
at the top, with Back / Next / Skip / Finish buttons. Keep the existing
`SetupWizard` class structure — add steps as internal methods.

The wizard must pass `InventoryService` and `PrinterService` instances in
for steps 2 and 3. Update `run_setup_wizard_if_needed()` signature:

```python
def run_setup_wizard_if_needed(db, parent, inventory_service=None, printer_service=None):
```

Update `main.py` to pass them.

### Task 3 — app.py header: show tenant brand

The header must show:
- Tenant logo (from `company_logo_path` setting, fall back to `config.LOGO_PATH`)
- Tenant company name (from `company_name` setting, fall back to `config.COMPANY["name"]`)
- Tenant subtitle below the name (from `app_subtitle` setting, fall back to
  `config.COMPANY["subtitle"]`)

Add `_resolve_tenant_logo()` and `_get_tenant_name()` private helpers.
Logo image is already loaded with PIL — just change the source path.

### Task 4 — login_dialog.py: show tenant brand

Same pattern: read `company_name` and `company_logo_path` from settings,
fall back to config. The `db=` kwarg already exists (from Phase 1 or prior
work) — use it.

### Task 5 — pdf_service.py: tenant body + Abaad footer

`_load_company()` must read from `db.get_all_settings()`:
- `company_name` → document header title
- `company_phone`, `company_address` → contact block
- `company_logo_path` → header logo (resolve relative to `PROJECT_ROOT`,
  fall back to `config.LOGO_PATH`)
- `currency_symbol` → all monetary formatting

Add a footer to every generated document (quote, receipt, invoice):
```
[page bottom]  Generated by Abaad ERP                    Page N of M
```
Small font, `Colors.TEXT_MUTED` equivalent in reportlab (grey).
Use `config.PDF_FOOTER_CREDIT` for the text — never hardcode the string.
