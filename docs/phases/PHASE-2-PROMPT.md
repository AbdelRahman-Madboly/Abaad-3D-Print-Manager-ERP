## Goal

Make "any 3D-printing business can deploy this fresh" actually true.
Every hardcoded piece of company identity moves to the `settings` SQLite table.
The first-run wizard is extended to capture a complete initial setup.
The app header, login screen, and all generated PDFs show the tenant's brand.

---

## The two-identity rule (read before touching anything)

```
PRODUCT  = Abaad (the app/maker)  → stays in config.py → NEVER change per customer
TENANT   = the business owner     → lives in settings table → configured at first run
```

Correct examples:
- Window title: `"Abaad ERP v5.0"` (product) ✓
- App header: `"Al-Nour 3D Printing"` + their logo (tenant) ✓
- PDF footer: `"Generated by Abaad ERP"` (product credit) ✓
- PDF body: tenant name, logo, phone, address ✓

### Task 3 — Extend the setup wizard to 4 steps

The current wizard (Step 1) captures company name, address, phone, tagline,
social, app subtitle, currency, rate per gram, and logo. Add 3 more steps.

**Step structure:** add a step indicator label (`"Step N of 4"`) at the top
of the wizard and Back / Next / Skip / Finish buttons. Keep the existing
`SetupWizard` class — add steps as internal `_build_step_N()` methods called
from a central `_show_step(n)` method that clears and rebuilds the card area.

Update `run_setup_wizard_if_needed()` signature to accept services:
```python
def run_setup_wizard_if_needed(db, parent, inventory_service=None, printer_service=None):
```

Update `main.py` to pass them — build `InventoryService` and `PrinterService`
before calling the wizard, the same way they're built for the main app.

**Step 2 — Initial filament inventory:**

Show a grid of the default colors from `config.DEFAULT_COLORS`. For each
color, a label + number entry (weight in grams, default `0`). User enters
how much of each color they currently have.

On "Next": for each color where grams > 0, call
`inventory_service.add_spool({...})` to create a spool record. Skip colors
with 0. "Skip step" leaves inventory empty.

**Step 3 — First printer:**

Fields pre-filled from `config.py` constants:
- Printer name — default `"Printer 1"`
- Purchase price — default `config.DEFAULT_PRINTER_PRICE`
- Expected lifetime kg — default `config.DEFAULT_PRINTER_LIFETIME_KG`
- Electricity rate EGP/hr — default `config.ELECTRICITY_RATE`
- Nozzle cost — default `config.NOZZLE_COST`
- Nozzle lifetime grams — default `config.NOZZLE_LIFETIME_GRAMS`

On "Next": call `printer_service.add_printer({...})`.
"Skip step" is fine — printers can be added later in the Printers tab.

**Step 4 — Cost defaults review:**

Show current business defaults and let user confirm or change:
- Selling rate per gram (from Step 1 or config default)
- Material cost per gram — `config.DEFAULT_COST_PER_GRAM`
- Deposit % — `DEFAULT_SETTINGS["deposit_percent"]`
- Quote validity days — `DEFAULT_SETTINGS["quote_validity_days"]`

"Finish Setup" button saves all to settings table and sets `setup_complete = "1"`.
