## What this project is

**Abaad ERP** — a desktop ERP for 3D-printing service businesses.
Stack: **Python 3.10+ / Tkinter / SQLite**, `reportlab` for PDFs,
optional `matplotlib` / `Pillow` / `pytesseract`.

Primary platform: **Ubuntu 24.04**. Secondary: **Windows 10/11**.
Distribution: Python venv + launcher script. No Docker (Tkinter needs a display server).

---

## Two identities — never confuse them

| Identity | What it is | Where it lives | Example |
|----------|-----------|----------------|---------|
| **Product** | Abaad (the app/maker) | `config.py` constants | Window title, About screen, PDF footer credit |
| **Tenant** | The customer's own business | `settings` SQLite table | App header, login screen, PDF body |

A generated invoice shows the **tenant's** logo and name at the top, and a small
`"Generated by Abaad ERP"` credit at the bottom. That is the full branding model.

## Architecture rules — never break these

1. **UI tabs** (`src/ui/tabs/*`) call services only — no SQL, no business logic.
2. **Services** (`src/services/*`) contain all business rules.
3. **DatabaseManager** (`src/core/database.py`) returns plain `dict`s only —
   services convert to/from dataclasses in `src/core/models.py`.
4. **`config.py`** holds product constants and fallback defaults only.
   User-editable values live in the `settings` SQLite table.
5. **No generic raw-SQL methods** (`execute_query`, `execute_update`, etc.)
   on DatabaseManager — use/extend typed methods.
6. **Scope discipline**: keep changes inside the current phase brief.
   Note out-of-scope issues in the completion report, never fix inline.
7. **Tests always green**: every change must keep `pytest` passing.
   Add/update tests for anything you fix or add.

---

## DB abstraction rule (for future Postgres migration)

All SQL lives in `DatabaseManager` typed methods. Services never build SQL strings.
The schema uses `is_deleted` (not `deleted`) for soft deletes on orders.

## How to run

```bash
cd ~/projects/products/Abaad-3D-ERP

# Run the app
python3 main.py

# Run tests (always from repo root, using conda base or venv with pytest)
python3 -m pytest -q

# Run a single test file
python3 -m pytest tests/test_order_service.py -v
```

**Dev environment note:** no venv exists yet (Phase 5 adds it). Run tests
with `python3 -m pytest` using the conda base env which has pytest installed.
After Phase 5, use `.venv/bin/python -m pytest`.

```
fix/*     → bug fixes
```

```bash
gh pr create --base develop --fill
gh pr merge --merge --delete-branch
git checkout develop && git pull
```

`fix` / `feat` / `refactor` / `chore` / `docs` / `test` / `remove`
