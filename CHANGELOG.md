## [6.0.0] ‚Äî 2026-06-17

### Added

**Phase 10 (React full-stack UI)**
- FastAPI bridge (`api/`) ‚Äî 28 authenticated REST endpoints across 9 routers
  (auth, orders, customers, filament, printers, expenses, failures, settings, dashboard)
- In-memory Bearer-token session auth backed by the existing Python service layer
- React 18 + TypeScript + Vite 6 frontend (`frontend/`) with shadcn/ui components,
  Tailwind CSS v4, and Recharts for dashboard charts
- Isometric 3D cube favicon (SVG, Abaad blue palette)
- Collapsible sidebar navigation; show/hide password toggle on login
- `frontend/src/lib/api.ts` ‚Äî typed fetch wrapper with automatic Bearer-token injection
- `frontend/src/lib/auth-context.tsx` ‚Äî React AuthContext + `useAuth()` hook
- All 10 screens wired to live data: Login, Dashboard, Orders, Customers,
  Filament, Printers, Expenses, Failures, Settings, Setup Wizard
- Authenticated PDF download via blob URL trick (`downloadFile()`)
- `make dev` target ‚Äî starts uvicorn (:8000) and Vite (:5173) concurrently

### Changed
- `make dev` is now the primary entry point; `python3 main.py` still runs the legacy Tkinter UI
- Phase docs reorganised into `docs/phases/Prompt/` and `docs/phases/Report/`

## [5.0.0] ‚Äî 2026-06-17

### Added

**Phases 0‚Äì4 (core foundation)**
- Unified Dashboard tab with Action Center, Headline Numbers, Charts,
  Printer Utilization, and Detailed Breakdowns sections
- First-run setup wizard (4 steps: company info, filament, printer, cost defaults)
- Per-printer print job tracking on order status transition to "Ready"
- `get_orders_needing_attention()` ‚Äî overdue Ready orders + unpaid delivered orders
- `get_printers_needing_maintenance()` ‚Äî nozzle wear alerts
- Tenant brand system ‚Äî company identity configurable via `settings` SQLite table
- App header shows company name and subtitle from settings
- Login screen shows company name and logo from settings
- `PDF_FOOTER_CREDIT` constant ‚Äî `"Generated by Abaad ERP"` on every PDF
- `get_print_items_totals()` ‚Äî single aggregate query for total weight/time printed
- GitHub Actions CI (lint with ruff + test with pytest on push/PR to develop/main)

**Phase 5 (launchers)**
- Ubuntu `.desktop` launcher (`abaad-erp.desktop`) for application menu integration
- Windows `Launch_App.bat` launcher
- `scripts/install.py` cross-platform venv + dependency installer

**Phase 6 (UI polish)**
- `Tooltip` widget ‚Äî hover tooltips on icon-only action buttons
- `ScrollableFrame` widget ‚Äî canvas-backed scrollable container; applied to
  Settings and Dashboard tabs
- Minimum window size (`900√ó700`) enforced at startup; default geometry `1200√ó750`
- Empty-state placeholder labels in all tabs when no data is present

**Phase 7 (PDF polish + documentation)**
- PDF footer drawn via `onFirstPage`/`onLaterPages` canvas callbacks ‚Äî appears on
  every page of multi-page PDFs, not just the first
- `default_cost_per_gram` added to `DEFAULT_SETTINGS` so callers can rely on
  `db.get_setting("default_cost_per_gram")` always returning a value
- Google-style docstrings added to all public methods across `src/core/`,
  `src/auth/`, `src/services/`, and `src/ui/` (app.py, widgets.py)
- PDF generation test suite (`tests/test_pdf_service.py`) using `pypdf>=5.0`

**Phase 8 (packaging)**
- PyInstaller Linux bundle ‚Äî `abaad-erp.spec` (6.x API: `PYZ(a.pure)`, no cipher);
  produces `dist/abaad-erp/` (~165 MB) running on Ubuntu 24.04 without Python
- `scripts/run_bundle_linux.sh` ‚Äî bundle launcher with existence check
- `Makefile` ‚Äî `venv / install / test / lint / build / run / run-bundle / clean`
- `pyinstaller>=6.0` added to dev extras in `pyproject.toml`
- `data/.gitkeep` ensures data directory is present in bundle for DB creation

### Fixed
- `auth_manager.py` no longer prints `admin / admin123` to stdout (now `log.info`)
- Settings tab DB access (was calling nonexistent `execute_query`)
- PDF service company info loading (was calling nonexistent `get_settings_by_keys`)
- Analytics aggregation methods (monthly revenue, order status, filament usage)
- Status bar filament key (`available_weight_g` not `total_available`)
- `total_weight_printed` / `total_time_printed` always returning 0 (fixed via
  `get_print_items_totals()` aggregate, not per-order property sum)
- Setup wizard now fires on first launch (was never wired into boot sequence)
- `config.DEFAULT_SETTINGS` missing required keys (caused crash on first run)
- `COMPANY` dict neutralized ‚Äî no hardcoded business contact info in source
- `pyproject.toml` build backend corrected (`setuptools.build_meta`)
- `generate_text_receipt()` hardcoded `"EGP"` ‚Äî now reads `currency_symbol` from
  order context dict, falling back to `"EGP"` only when not set
- `default_cost_per_gram` absent from `DEFAULT_SETTINGS` ‚Äî caused `KeyError` on
  fresh installs before wizard completion

### Removed
- v4 migration script and tests
- Hardcoded Abaad business contact info from `config.COMPANY` (phone, address, social)
- Separate Stats and Analytics tabs (merged into Dashboard)
- Dead `analytics_tab.py` (superseded by `dashboard_tab.py`)
- `requirements.txt` (all deps now declared in `pyproject.toml`)
